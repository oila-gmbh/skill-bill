# Skill Bill demo — capture & encode instructions

This document is the recipe for producing the front-door motion demo described in
[skill-bill-demo-storyboard.md](skill-bill-demo-storyboard.md). The binary is
produced **out-of-band** (it is not generated by any build step in this repo) and
then committed to swap out the static placeholder.

## Canonical output filename

> **Capture to `docs/assets/skill-bill-demo.gif`.**

The README currently embeds the placeholder
[`skill-bill-demo-placeholder.svg`](skill-bill-demo-placeholder.svg). When the real
capture is ready:

1. Produce the recording per the steps below and save it as
   **`docs/assets/skill-bill-demo.gif`**.
2. In `README.md`, change the demo image source from
   `docs/assets/skill-bill-demo-placeholder.svg` to
   `docs/assets/skill-bill-demo.gif`.
3. Keep the storyboard and these instructions linked from the README so the asset
   stays reproducible.
4. Leave the placeholder SVG in the repo (it documents intent and is a safe
   fallback); do not delete it.

If the final recording is a longer-form clip where MP4 quality/size matters more
than autoplay simplicity, capture to **`docs/assets/skill-bill-demo.mp4`** instead
and embed it with an HTML `<video>` tag (see "GIF vs MP4" below). Pick one canonical
filename and use it consistently — the default and recommended choice is the GIF.

## Target spec

- **Resolution:** capture at 1920×1080, downscale to **1200px wide** for the embed
  (matches the hero image width). Keep 16:9.
- **Length:** **18–24 seconds** (per the storyboard). Hard ceiling 30s.
- **Frame rate:** 12–15 fps is plenty for a terminal demo and keeps GIF size down.
- **File size budget:** aim for **≤ 8 MB** for a GIF so GitHub renders it inline
  without lazy-loading hiccups. If you cannot get under budget at acceptable
  quality, switch to MP4.

## GIF vs MP4

GitHub renders **both** GIF (via `![]()` markdown) and MP4 (via an HTML `<video>`
tag or by uploading to an issue/PR and using the resulting asset URL) inline in
rendered Markdown.

- **GIF** — simplest for short loops (≤ ~10s) and for a Markdown image embed that
  needs no HTML. Larger files, no audio, capped color palette. **Default choice
  for this demo.**
- **MP4 / `<video>`** — markedly smaller and higher quality for longer or
  higher-motion demos; supports controls. Prefer it if the clip grows past ~10s or
  the GIF blows the size budget. Embed example:

  ```html
  <video src="docs/assets/skill-bill-demo.mp4" controls width="1200"
         muted playsinline></video>
  ```

## Suggested tooling

Recording:

- **asciinema** + **agg** (`asciinema rec` → `agg demo.cast docs/assets/skill-bill-demo.gif`)
  — best for pure-terminal flows; crisp text, tiny files.
- **terminalizer** — terminal recorder with built-in GIF export and theming.
- **OBS Studio** — for the desktop-app alternate flow, or any GUI capture; export
  MP4, then convert.

Encode / convert (GIF):

```bash
# From an MP4/MOV master to an optimized, palette-tuned GIF
ffmpeg -i master.mov -vf "fps=15,scale=1200:-1:flags=lanczos,split[s0][s1];\
[s0]palettegen[p];[s1][p]paletteuse" -loop 0 docs/assets/skill-bill-demo.gif

# Further shrink if over budget
gifsicle -O3 --lossy=60 docs/assets/skill-bill-demo.gif -o docs/assets/skill-bill-demo.gif
```

Encode (MP4, if chosen):

```bash
ffmpeg -i master.mov -vf "scale=1200:-1:flags=lanczos" -c:v libx264 \
  -pix_fmt yuv420p -crf 23 -movflags +faststart -an docs/assets/skill-bill-demo.mp4
```

## Honesty rules

- Record a **real** run against a real small spec. Do not stage fake output.
- You may fast-forward model latency, but do not reorder or invent pipeline phases.
- Keep secrets, tokens, and private identifiers out of frame.
