# Python UX Accessibility Review

Focus on accessibility and user experience in Python-rendered or Python-generated interfaces.

## Review Focus

- Semantic structure: headings, landmarks, labels, table headers, form grouping, error summaries, and status messages in server-rendered templates or generated reports.
- Forms and validation: field-level errors, focus targets, required/optional clarity, recovery paths, preserved input, and actionable copy.
- Keyboard and focus: modal/dialog behavior, admin actions, custom controls, skip/navigation affordances, and focus order after updates.
- Assistive technology: ARIA only where needed, screen-reader announcements for dynamic fragments, alt text, accessible names, and contrast-sensitive content generated by templates.
- Localization-sensitive UX: pluralization, date/time/number formatting, timezone clarity, text embedded in generated images/reports, and copy that can be translated safely.

## Findings Standard

Report issues that block understanding, operation, recovery, or assistive-technology use. Keep findings tied to the rendered output the Python code produces.
